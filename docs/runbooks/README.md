# Runbooks

Use this directory for step-by-step operating procedures.

## Runbook Rules

- Each runbook should have a clear trigger or purpose.
- Steps should be ordered, explicit, and easy to execute under time pressure.
- Include prerequisites, rollback notes, and verification steps when relevant.

## Suggested Naming

- `bootstrap-dev-environment.md`
- `publish-macos-build.md`
- `recover-failed-generation-job.md`
