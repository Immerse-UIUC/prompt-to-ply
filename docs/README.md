# Documentation Map

This repository uses `docs/` as the home for project documentation.

## Directories

- `architecture/`: canonical system design, architecture revisions, and related diagrams
- `api/`: service contracts, message schemas, and interface notes
- `qa/`: quality strategy, acceptance criteria, and validation plans
- `prompts/`: prompt assets, prompt fixtures, and evaluation inputs
- `research/`: experiments, references, and comparative findings
- `operations/`: environments, deployment, monitoring, and support policies
- `runbooks/`: procedural step-by-step operating guides
- `decisions/`: durable architectural and product decisions
- `worklog/`: next tasks, sequencing, and execution notes
- `agents/`: agent-facing workflow and repo guidance

## Conventions

- Put canonical, durable documents in the most specific folder available.
- Keep active work tracking in `docs/worklog/`.
- Keep durable decisions in `docs/decisions/`.
- Keep architecture changes aligned with `docs/architecture/ARCHITECTURE-v2.md`.
